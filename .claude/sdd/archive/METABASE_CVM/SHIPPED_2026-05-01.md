## Lições Aprendidas

### Processo

**Automação via `/iterate` foi a decisão certa — e foi rápida.**

O DESIGN v1.0 foi criado com criação manual (UI). A troca para script Python via `/iterate` levou menos de 15 minutos de refatoração e converteu um processo de ~30 min manual em `make metabase-setup-cvm`. A velocidade de mudança valida o modelo SDD: o DESIGN como contrato permite iterar sem risco de regressão nos artefatos existentes.

**Padrão de automação Metabase estabelecido para domínios futuros.**

O Makefile agora tem `metabase-setup-bcb` (BCB) e `metabase-setup-cvm` (CVM) — padrão uniforme entre domínios. Próximos domínios seguem o mesmo template: `scripts/setup_metabase_{domain}.py` + `make metabase-setup-{domain}`.

### Técnico

**JOIN pré-calculado na Gold elimina complexidade na camada de visualização.**

O fato de `taxa_anual_bcb` e `acumulado_12m_ipca` já estarem em `gold_cvm.fundo_mensal` significa que todos os 13 cards do Dashboard 3 (cross-domain) usam uma única tabela. Isso valida a decisão de materializar JOINs na camada Gold em vez de deixá-los para ferramentas de BI — menos complexidade, menos pontos de falha.

**IDs negativos na API Metabase são comportamento esperado.**

`PUT /api/dashboard/{id}/cards` exige IDs negativos para posições novas (`"id": -1, -2, ...`). Documentar isso evita horas de debugging em implementações futuras.

**`requests` precisa estar no ambiente Python.**

O script usa `requests` como dependência externa. Em um projeto com `uv`, precisar adicionar `uv add requests` antes de rodar o script é um passo que não estava no DEFINE original. Registrar dependências de scripts na seção de pré-requisitos do SETUP é obrigatório.

### Ferramentas

**`-include .env` + `export` no Makefile simplifica todos os targets.**

O Makefile agora usa `-include .env\nexport` no topo, o que torna desnecessário o `@set -a && . ./.env && set +a &&` em cada target. Isso foi adicionado pelo usuário paralelamente ao /iterate — padrão mais limpo para todos os targets futuros.
